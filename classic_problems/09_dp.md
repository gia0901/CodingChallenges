# 9. Dynamic Programming ★★★☆☆

Khó nhất trong nhóm phổ biến. Chìa khóa: **định nghĩa state rõ ràng** + **công thức chuyển tiếp** + **base case**. Ba khuôn: 1D, 2D (hai chuỗi/lưới), knapsack.

## Các pattern cốt lõi (skeleton C++)

### Pattern A — 1D DP
```cpp
vector<int> dp(n + 1);
dp[0] = base;
for (int i = 1; i <= n; ++i)
    dp[i] = /* hàm của dp[i-1], dp[i-2], ... */;
return dp[n];
```

Climbing Stairs — mỗi bậc gộp từ 2 bậc trước:

```
 dp:  1  1  2  3  5  8      dp[i] = dp[i-1] + dp[i-2]
 i :  0  1  2  3  4  5      (chỉ phụ thuộc 2 ô trước -> ép Space O(1))
```

### Pattern B — 2D DP (hai chuỗi / lưới)
```cpp
vector<vector<int>> dp(m + 1, vector<int>(n + 1, 0));
for (int i = 1; i <= m; ++i)
    for (int j = 1; j <= n; ++j)
        dp[i][j] = (a[i-1] == b[j-1]) ? dp[i-1][j-1] + 1
                                      : max(dp[i-1][j], dp[i][j-1]);
```

LCS "ace" vs "abcde" — mỗi ô nhìn 3 ô (trên, trái, chéo trên-trái):

```
      ""  a  b  c  d  e
   "" 0   0  0  0  0  0
   a  0   1  1  1  1  1
   c  0   1  1  2  2  2
   e  0   1  1  2  2  3   <- đáp = 3 ("ace")
```

### Pattern C — 0/1 Knapsack (1D tối ưu space)
```cpp
vector<int> dp(W + 1, 0);
for (int i = 0; i < n; ++i)
    for (int w = W; w >= wt[i]; --w)      // duyệt NGƯỢC: mỗi item dùng ≤ 1 lần
        dp[w] = max(dp[w], dp[w - wt[i]] + val[i]);
```

## Bài cốt lõi (7 bài)

| # | Bài | Pattern | Complexity mục tiêu |
|---|-----|---------|---------------------|
| 1 | Climbing Stairs | A | Time O(n), Space O(1) |
| 2 | House Robber | A | Time O(n), Space O(1) |
| 3 | Coin Change | knapsack (unbounded) | Time O(n·amount) |
| 4 | Longest Increasing Subsequence | A | O(n²) hoặc O(n log n) |
| 5 | Longest Common Subsequence | B | Time O(m·n) |
| 6 | Unique Paths | B (lưới) | Time O(m·n) |
| 7 | Partition Equal Subset Sum | C (subset sum) | Time O(n·sum) |

### 1. Climbing Stairs
> `dp[i] = dp[i-1] + dp[i-2]` (Fibonacci). Nền tảng.
- Đã có: [leetcode/dynamic_programming/70_climbing_stairs.cpp](../leetcode/dynamic_programming/70_climbing_stairs.cpp).

### 2. House Robber
> Không cướp 2 nhà liền kề: `dp[i] = max(dp[i-1], dp[i-2] + a[i])`.

### 3. Coin Change
> Số coin ít nhất tạo `amount`. Unbounded knapsack: `dp[x] = min(dp[x], dp[x-coin] + 1)`, duyệt `x` **tăng**.

### 4. Longest Increasing Subsequence
> O(n²): `dp[i] = max(dp[j]+1)` với `j<i, a[j]<a[i]`. O(n log n): mảng `tails` + `lower_bound`.

### 5. Longest Common Subsequence
> Pattern B (skeleton ở trên). Nền cho edit distance, diff.

### 6. Unique Paths
> Số đường từ góc trên-trái xuống dưới-phải: `dp[i][j] = dp[i-1][j] + dp[i][j-1]`.

### 7. Partition Equal Subset Sum
> Chia mảng thành 2 phần tổng bằng nhau ⇔ subset sum = `total/2` (0/1 knapsack boolean).

## Bài luyện thêm (interview hay gặp)

| Bài | Khuôn / ý tưởng | Complexity |
|-----|-----------------|------------|
| Min Cost Climbing Stairs | 1D DP, `dp[i]=cost[i]+min(dp[i-1],dp[i-2])` | O(n) |
| House Robber II | nhà xếp vòng → chạy 2 lần (bỏ đầu / bỏ cuối) | O(n) |
| Maximum Product Subarray | giữ cả `max` và `min` chạy (số âm đảo dấu) | O(n) |
| Word Break | 1D DP trên chuỗi + set từ điển | O(n²) |
| Decode Ways | 1D DP, mỗi bước chọn 1 hoặc 2 chữ số | O(n) |
| Coin Change II (đếm cách) | unbounded knapsack **đếm**, coin vòng ngoài | O(n·amount) |
| Longest Palindromic Subsequence | 2D DP theo độ dài đoạn | O(n²) |
| Edit Distance | 2D DP (insert/delete/replace) — khó hơn | O(m·n) |

- **Min Cost Climbing Stairs / House Robber II**: biến thể 1D của bài đã có. House Robber II "vòng tròn": nhà đầu và cuối kề nhau → chạy House Robber trên `[0..n-2]` và `[1..n-1]`, lấy max.
- **Maximum Product Subarray**: khác Kadane ở chỗ số âm × âm = dương → phải giữ **cả `maxSoFar` và `minSoFar`**, khi gặp số âm thì hoán đổi vai trò.
- **Word Break**: `dp[i]` = cắt được `s[0..i)` không? `dp[i] = OR các dp[j] && s[j..i) ∈ dict`. Đưa dict vào `unordered_set` để tra O(1).
- **Decode Ways**: đếm cách giải mã "12" → 'A'..'Z'. `dp[i]` gộp từ 1 chữ số (`s[i-1]≠'0'`) và 2 chữ số (`10..26`). Bẫy số 0.
- **Coin Change II**: đếm **số cách** (không phải min coin). Để không đếm trùng hoán vị: **coin ở vòng lặp ngoài**, amount vòng trong.
- **Edit Distance / LCS / LPS**: cùng họ 2D DP (Pattern B). Nắm LCS trước rồi mở rộng; Edit Distance thêm 3 lựa chọn (thêm/xóa/thay).

## Vì sao thứ tự duyệt knapsack quyết định đúng/sai

Bản 1D dùng chung mảng `dp` cho item hiện tại và item trước:

- **0/1 (mỗi item ≤ 1 lần) → duyệt `w` NGƯỢC**: khi tính `dp[w]`, `dp[w-wt]` vẫn là giá trị
  của các item **trước đó** (chưa cập nhật ở vòng này) → không dùng lại chính item hiện tại.
- **Unbounded (dùng lại tùy ý) → duyệt `w` XUÔI**: `dp[w-wt]` đã được cập nhật ở chính vòng
  này → cho phép cộng item hiện tại nhiều lần.

Sai chiều = sai bài. Đây là bẫy DP kinh điển.

## Pitfall C++ hay gặp

- **Thứ tự duyệt**: 0/1 knapsack duyệt trọng số **ngược**; unbounded duyệt **xuôi** — sai chiều là sai kết quả.
- **Base case & khởi tạo**: `INT_MAX` sentinel dễ **overflow** khi `+1` → kiểm tra trước khi cộng.
- **Định nghĩa state mơ hồ** là lỗi gốc — viết rõ `dp[i]` nghĩa là gì trước khi code.
- Có thể ép **O(1)/O(n) space** bằng rolling array khi chỉ phụ thuộc hàng/cột trước.
